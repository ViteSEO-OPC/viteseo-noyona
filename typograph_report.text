# Noyona Theme — Typography & Design-Token Consistency Audit (read-only)

---

## 1. Global Typography Tokens

### `theme.json`
| What | Value | Source |
|---|---|---|
| Font family — sans (default) | `"Proxima Nova", system-ui, -apple-system, "Segoe UI", sans-serif` | `theme.json:13` |
| Font family — serif (h1 only) | `"Noto Serif SemiCondensed", "Noto Serif", "Times New Roman", serif` | `theme.json:18` |
| Body size | `clamp(16px, 1.4vw, 20px)` | `theme.json:25` |
| `h-3` size | `clamp(18px, 2vw, 24px)` | `theme.json:26` |
| `h-2` size | `clamp(32px, 4.2vw, 55px)` | `theme.json:27` |
| `h-1` size | `clamp(44px, 6vw, 96px)` | `theme.json:28` |
| Body weight | `400` | `theme.json:42` |
| Body color (text) | `#333333` | `theme.json:38` |
| Color palette | one entry only: `ink = #333333` | `theme.json:31-35` |

### Global rules in `style.css`
- `:root` defines `--footer-bg: #EBAAAC`, `--footer-text: #1a1a1a`, `--footer-btn-bg: #E199A4` (`style.css:19-39`).
- `body` re-declares the same family/size/weight/line-height as `theme.json` (`style.css:112-117`).
- `h1`, `h2`, `h3` rules in `style.css:119-138` duplicate `theme.json`'s element styles.

### Font-face declarations
- `assets/css/fonts.css` — six `@font-face` declarations for Proxima Nova at weights **300, 400, 600, 700, 800, 900**, all `font-display: swap`. So the site actually loads weights 800/900 even though only a handful of selectors use 800.

### Mismatches between `theme.json` and CSS
- Body family/size/weight is declared in **both** sources (redundant — one source of truth would be cleaner).
- `h4`, `h5`, `h6` have **no global declaration** in `theme.json` *or* `style.css`. Each component invents its own treatment.
- Palette has 1 color (`ink`) but the actual CSS file uses 280+ distinct hex values (see §14).

## 16. Top 10 Biggest Typography/Design Inconsistencies

1. **`#E199A4` has 35+ near-duplicate variants** — most painful issue. Brand pink, footer pink, account pink, hover pink, raspberry, hot-pink, soft-pink panels — at least 35 hexes do the same job. Multiple files use *different* hexes for the *same UI role* (footer bg, hover state, accent text).
2. **9+ near-black text colors** (`#000`, `#111`, `#1a1a1a`, `#1b1b1b`, `#1d1d1d`, `#1f1f1f`, `#212121`, `#222`, `#333`, `#0f1728`, `#101828`, `#121212`). No rule for when to use which.
3. **`letter-spacing: -5%` is invalid CSS** and silently dropped — `header.css:301` and `header.css:378`. Nav links are missing their intended tightening.
4. **No global `h4`/`h5`/`h6` declaration** — every component invents its own. Result: `h5` is `clamp(24px,1.8vw,32px)` in one place and `18px` in another (same selector, scoped twice).
5. **Account section uses ~15 distinct `calc(--body * N)` multipliers** for sizing — there is no scale, just per-component arithmetic. Two `<h3>` elements are styled at h2 size.
6. **Five different border-radii for CTAs** (`10px`, `14px`, `16px`, `26px`, `999px`) — pill, rounded, half-rounded all coexist.
7. **`footer.css` defines the same selectors twice** (original 3-308 + redesign 405-765). Newsletter h2 has two sizes; nav has two letter-spacings (`0.02em` vs `-1px`); button has two backgrounds, colors, and radii.
8. **Three button "hover" patterns** for the primary pink CTA — `#cc848f`, `#d30b57`/`#c70a52`, and `filter:brightness(0.97)` — all on what is the same conceptual button.
9. **Five different fallback chains for "Proxima Nova"** across files. `header.css` even chains the literal name AND the WP CSS variable. `inc/shortcodes.php:3508` ships Arial in invoices.
10. **Two heading weights diverge from the global 700 standard**: `.noyona-shop-title.wp-block-query-title` uses **800** (`style.css:955`), `.hero-banner` h1 uses **500** (`blocks/hero-banner/style.css:78,105`), `coming-soon`/`terms` h1 use **600**. Combined with the 6 `@font-face` weights loaded (300/400/600/700/800/900), the site is paying for fonts it barely uses.

> Honorable mentions: `font-size:0` icon-only hacks in mini-cart/PDP, `!important` color overrides in account buttons and mini-cart drawer, the inline `style="color:#E199A4;font-weight:600;font-size:14px"` on `.noyona-shop-view-label` duplicated across 7 templates instead of styling via CSS.
